# When Stuck

Dispatch table for total impasse.

## Diagnosis → Action

| Situation | Action |
|-----------|--------|
| Tried 3+ approaches, all failed | Switch technique — use a different problem-solving method from this skill |
| Same error repeating | Step back and question the frame — the error might be a symptom, not the cause |
| All solutions feel wrong | Check if the problem statement is correct — you may be solving the wrong problem |
| Deadline pressure mounting | Simplify scope first — cut to the absolute minimum that delivers value |
| Too many options, can't decide | Use the inversion technique — eliminate the worst options first |
| Problem is too abstract | Find a concrete example — solve one specific case, then generalize |
| Problem is too concrete | Abstract it — strip domain details and look for the underlying pattern |

## Escalation Path

1. Try a different technique (5 min)
2. Write down what you've tried and what you've learned (forces clarity)
3. Question the problem statement (is this the right problem?)
4. Ask for help with a specific, well-framed question (not "I'm stuck")
5. Sleep on it (if timeline allows — fresh perspective often unlocks)

## Anti-patterns

- Retrying the same approach with minor variations
- Adding more complexity hoping it fixes the issue
- Asking for help without describing what you've already tried
