# Workflow: CRO Copy

Conversion rate optimization principles for copy. Apply when copy's primary goal is measurable conversion improvement on a specific page or asset.

---

## Headlines

| Principle | Guidance |
|-----------|----------|
| 4-U formula | Every headline should be Useful, Urgent, Unique, and Ultra-Specific — score each dimension |
| Above-fold guarantee | The primary benefit must be visible before any scrolling on desktop and mobile |
| Benefit-first structure | Lead with what the reader gains; explain how it works after |
| Specificity over vagueness | "Cuts reporting time by 60%" converts better than "saves time" |
| A/B variable | Headline is the single highest-leverage element to test first |
| Audience qualifier | Identify the right reader in the first 6 words ("For B2B teams that...") |
| No cuteness above fold | Clever headlines that obscure the value proposition reduce conversion |
| Sub-headline role | Subheadline should expand on the headline; not restate it |

---

## CTAs and Forms

| Principle | Guidance |
|-----------|----------|
| First-person CTA text | "Start my free trial" outperforms "Start your free trial" |
| Benefit in button copy | Button text names what they get, not the action they take |
| Maximum form fields | 3 fields for lead gen; only ask what you need right now |
| Progress indicators | Multi-step forms show step X of Y; reduces abandonment |
| Micro-commitment ladder | Start with email only, add qualifying data in step 2 |
| One primary CTA per section | Competing CTAs cancel each other out |
| Secondary CTA pattern | Primary: "Book a demo" + Secondary: "Watch a 3-min overview" |
| CTA contrast | Button must be the highest-contrast element in its section |
| Microcopy placement | Place risk reversal within 2 lines of every CTA |

## Cognitive Psychology Principles

| Principle | Application to Copy |
|-----------|-------------------|
| Loss aversion | Frame benefits as avoiding loss before emphasizing gain ("Stop losing leads to slow follow-up") |
| Anchoring | Show a higher-priced tier before the target tier; show full value before the discounted price |
| Social proof cascade | Stack proof: number of users → logo bar → named testimonial with metric → star rating |
| Commitment and consistency | Use micro-commitments early (email for a free resource) before asking for bigger commitment |
| Message match | Copy on an ad and the landing page it links to must use the same language and promise |
| Cognitive ease | Grade 6 reading level reduces friction; complexity creates hesitation |
| Peak-end rule | Last thing a reader sees (CTA, P.S., closing statement) carries disproportionate weight |
| PAS bias amplification | Agitation phase should feel personal and specific — generic pain descriptions don't agitate |
| Paradox of choice | Reduce options; too many plan tiers or CTAs reduce conversion |
| Reciprocity | Give something valuable before asking; free resource → offer converts better than cold ask |

## Pricing and Urgency

| Principle | Guidance |
|-----------|----------|
| Price anchoring | Display higher tier or original price first; makes target price feel smaller |
| Genuine urgency only | Fake countdown timers damage trust; use real deadlines or real scarcity |
| Urgency framing | "Offer ends Friday" converts better than "Limited time only" |
| Per-unit pricing | "$2/day" feels smaller than "$60/month" — use the frame that reduces psychological cost |
| Value stack before price | Show the full stack of what they get before revealing the number |
| Risk reversal placement | Put guarantee language directly adjacent to price; not in the FAQ |
| Trust signal clustering | Group: social proof + guarantee + privacy note near every price point |
| Free trial vs freemium | Free trial (time-limited) creates urgency; freemium creates habit — choose intentionally |

## Layout and UX (Copy Implications)

| Principle | Copy Implication |
|-----------|-----------------|
| F-pattern reading | Lead sentences and headlines carry the conversion load; endings are rarely read |
| Mobile thumb zone | Primary CTA must be reachable with thumb on mobile (avoid top-center of screen) |
| White space | Short paragraphs (1–3 sentences) increase scan and perceived clarity |
| Scroll depth signals | Critical proof and CTAs at 25%, 50%, 75%, and 100% scroll depth |
| Video above fold | Explainer video on landing page increases conversion 80% on average (A/B test it) |
| Long-form vs short-form | Higher price / higher consideration = longer copy justified |
| Section headers as scanners | Every section header should communicate value, not just label the section |
| Testimonial placement | Place testimonials directly before or after the element they address (price → pricing testimonial) |

## Testing Principles

| Principle | Guidance |
|-----------|----------|
| One variable per test | Isolate headline, CTA text, hero image, or social proof — never test combinations |
| Statistical significance | Run until 95% confidence; do not call tests early |
| Test highest-traffic first | Start with headline and hero CTA; most impact per test |
| Control exists | Always run against a control; never test two new variants against each other in a first test |
| Negative testing | Test removing elements (social proof, imagery) — simplification sometimes wins |
| Copy length test | Short vs long is a legitimate test; long copy wins on high-consideration, short on low |
| Post-conversion momentum | Thank-you pages are underused conversion surfaces; A/B a second offer or referral ask |
| Mobile vs desktop | Segment results; mobile and desktop audiences behave differently |

## Copy Quality Standards

| Standard | Target |
|----------|--------|
| Reading level | Grade 6 (Hemingway App score ≤6) for consumer and general B2B |
| Sentence length | Average under 14 words |
| Paragraph length | Maximum 3 sentences; 1–2 preferred |
| Jargon | Zero unexplained acronyms or product-internal terms |
| Passive voice | Under 10% of sentences |
| Claim specificity | Every quantified claim has a source or verifiable evidence |
| Funnel alignment | TOFU copy never uses BOFU urgency language; BOFU copy always includes proof |
| CTA count | One primary per section; two only when secondary is a significantly softer ask |
| Social proof specificity | Testimonials include: name, role, company, and a specific outcome metric |
| Value proposition clarity | A stranger should understand the offer in under 3 seconds |
