# Marketing Reporting

Turns metrics into decisions and workflows into repeatable steps. Use when building a marketing
report, dashboard, or SOP.

## When To Use

- Building a recurring (weekly/monthly) performance report
- Designing a dashboard or KPI tree
- Writing an SOP that standardizes a marketing workflow
- Defining UTM/tracking conventions

## Rules

1. Every metric carries a comparison (vs target and vs prior period) and a "so what" — a number
   with no interpretation is noise.
2. End every report with prioritized next actions tied to an owner; reporting exists to drive decisions.
3. Separate leading indicators (steerable now) from lagging ones (the outcome); lead the report
   with what the team can act on.
4. State attribution caveats; never imply causation from correlation, and never present a
   single-touch number as the whole truth.
5. Read-only and honest: pull real numbers; if data is missing or broken, show the gap rather than
   estimating silently.
6. SOPs are numbered, owned, and checklist-backed — they codify an existing good workflow, not an
   aspiration.

## Output Format

A report: executive summary → metrics by funnel stage/channel (vs target + prior) → interpretation
→ prioritized next actions. An SOP: purpose → owner → inputs → numbered steps → checklist →
failure modes.

## Anti-patterns

- Dumping metrics with no comparison or interpretation.
- Reports that end without a recommended action.
- Mixing leading and lagging indicators without labeling them.
- Implying a channel "caused" results from last-click attribution alone.
- SOPs written as prose paragraphs instead of numbered, checkable steps.

## Gotchas

- A weekly report nobody acts on is a standing tax — cut metrics that never change a decision.
- Vanity metrics (impressions, followers) feel good and steer badly; anchor on funnel and revenue metrics.
- Attribution windows and models quietly change the story — state them so trends stay comparable.
- Comparing to target without comparing to prior period (or vice versa) hides half the picture.
- An SOP rots the moment the workflow changes — date it and name an owner responsible for refresh.
