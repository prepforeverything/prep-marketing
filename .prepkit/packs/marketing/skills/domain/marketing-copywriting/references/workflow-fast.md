# Workflow: Fast Copy

Use for time-sensitive copy work where the goal is rapid, good-enough output for low-to-medium-stakes assets.

## When to Use This Workflow

| Use | Skip |
|-----|------|
| Social post (any channel) | Homepage redesign |
| Email subject line variations | Full landing page |
| Ad headline sets | Campaign positioning |
| Product description (single SKU or feature) | New product launch copy |
| Short email draft (under 200 words) | Brand voice definition |
| Event promotion copy | Case study or long-form |
| Bio or profile copy | High-ticket sales page |
| Notification or error message copy | Investor materials |

**Rule of thumb:** if the asset has a test-and-learn path, use Fast Workflow. If it will be seen by thousands and cannot be easily revised, use the full brief process.

---

## Step 1: Gather Context (2 minutes)

Answer these 5 questions before writing:

| Question | Input |
|----------|-------|
| Who is the reader? | |
| What is the one action they should take? | |
| What is the primary pain or desire this addresses? | |
| What is the channel and character/word limit? | |
| Is there existing copy to adapt or beat? | |

If `marketing-product-context` has been run, extract from sections 4 (pains), 6 (audience), and 9 (voice of customer). Skip gathering if it's already there.

---

## Step 2: Select Formula (30 seconds)

| Asset Type | Default Formula |
|-----------|----------------|
| Social post | PAS or AIDA (hook → problem → solution) |
| Ad headline | 4Us (useful, urgent, unique, ultra-specific) |
| Email subject line | Curiosity gap or benefit-driven |
| Short email | PAS |
| Product description | FAB |
| CTA button | Action + Benefit |

Refer to `copy-formulas.md` quick-reference table if unsure.

---

## Step 3: Draft (5–10 minutes)

Draft 3 versions minimum. For each version:

- Apply the selected formula
- Use one power word from `power-words.md` in the headline or opener
- Match the tone to the brand style profile from `writing-styles.md`
- Stay within channel limits from `channel-copy-norms.md`

**Draft format:**

```
Version A — [Formula used]
[Copy]
Rationale: [One sentence on why this should work]

Version B — [Alternative angle]
[Copy]
Rationale: [One sentence]

Version C — [Control / safer option]
[Copy]
Rationale: [One sentence]
```

---

## Step 4: Channel Adapt (2–5 minutes)

If the copy will run on multiple channels, adapt tone and length per channel norms. Do not copy-paste across channels.

| Channel | Adaptation Required |
|---------|-------------------|
| Email subject → LinkedIn post | Add context and a hook; remove urgency framing |
| LinkedIn post → X/Twitter | Cut to 240 chars; sharpen the hook |
| Landing page CTA → Email CTA | Add first-person phrasing |
| Ad headline → Social post | Expand to include the benefit story |

Reference `channel-copy-norms.md` for per-channel limits.

---

## Fast Copy Quality Check (30 seconds)

Before submitting, confirm:

- [ ] Is the primary benefit in the first line?
- [ ] Is the CTA a single clear action?
- [ ] Does it avoid internal jargon?
- [ ] Is it within channel length limits?
- [ ] Have you written at least 3 variants?
- [ ] Does one variant use a number for specificity?

---

## Fast Workflow Output Format

Deliver to stakeholders in this format:

```
Asset: [type and channel]
Goal: [one-line conversion goal]
Formula used: [name]

Option 1 (Recommended)
[Copy]
Why: [one sentence]

Option 2
[Copy]
Why: [one sentence]

Option 3
[Copy]
Why: [one sentence]

Suggested test: [what to A/B if testing is possible]
```
