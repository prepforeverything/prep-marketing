# Landing Page Copy

## Above-Fold Structure

| Element | Role | Guidance |
|---------|------|----------|
| Pre-headline | Context setter / audience qualifier | 1 line; "For [audience]" or "[Category]: [differentiator]" |
| Headline | Primary value statement | 6–12 words; benefit-first; no jargon |
| Subheadline | Expand and clarify | 1–2 sentences; add specificity or proof point |
| Hero image / video | Visual proof of outcome | Show the product in use or result state; not stock photos |
| Primary CTA | Conversion action | Benefit-first button text; one action only |
| Social proof element | Trust anchor | Logo bar, star rating, or one testimonial within 100px of CTA |

---

## Headline Formulas for Landing Pages

| Formula | Template | Example |
|---------|----------|---------|
| Outcome in timeframe | [Outcome] in [timeframe] | Hire your first engineer in 14 days |
| For [audience] who [desire] | For [audience] who [aspiration] | For founders who hate managing spreadsheets |
| The [category] that [differentiator] | The [category] that [unique claim] | The CRM that actually gets used |
| [Number] [things] to [outcome] | [N] [steps/tools] to [goal] | 3 steps to a landing page that converts |
| [Problem]? [Solution] | [Pain]? [Product] fixes it | Losing leads to slow follow-up? Automate it. |
| Do [outcome] without [pain] | [Achieve] without [friction] | Run campaigns without touching code |

## Benefits vs Features Mapping Table

Use this structure for every product feature listed on the page.

| Feature (what it does) | Advantage (why it's better) | Benefit (what they gain) |
|------------------------|----------------------------|-------------------------|
| Automated email sequences | No manual follow-up required | Never lose a lead to slow outreach again |
| Real-time analytics dashboard | See performance as it happens | Make decisions on live data, not last week's export |
| One-click integrations | Works with tools you already use | Up and running in under an hour |
| A/B testing built in | Test without a developer | Find your best variant in days, not months |
| GDPR compliance controls | Built to meet EU requirements | Ship in Europe without a compliance audit |

## How It Works Section Structure

Keep to 3–4 steps maximum. Each step follows: icon/number + title + one-line description.

| Step | Title pattern | Description pattern |
|------|--------------|---------------------|
| 1 | Connect / Set up / Import | [One action] in [timeframe or # clicks] |
| 2 | Configure / Customize / Select | [What they define or choose] |
| 3 | Launch / Go live / Run | [The moment of output] |
| 4 | Track / Optimize / Grow | [The ongoing benefit] |

## FAQ Section — Objection Mapping

Each FAQ answer should directly kill a hesitation, not just provide information.

| Common Objection | FAQ Answer Approach |
|------------------|-------------------|
| "Is it hard to set up?" | State exact setup time + steps required |
| "What if it doesn't work for us?" | Guarantee language + trial or refund policy |
| "Do I need technical skills?" | No-code / no-dev confirmation |
| "What does it integrate with?" | Named integration list + "and 200+ more" |
| "What does it cost?" | If pricing is hidden, explain the model clearly |
| "Is my data safe?" | Certification, encryption standard, compliance |
| "Will this work for my company size?" | Specific use cases for their segment |
| "What support do I get?" | SLA or human-support confirmation |

## Long-Form Page Flow

For high-consideration or high-ticket offers (SaaS, consulting, courses):

```
1. Hero — headline, subhead, CTA, social proof
2. Problem / Agitation — name the pain they're living with
3. Introduce the solution — what it is and why it's different
4. How it works — 3–4 steps
5. Features with benefits — FAB structure per feature
6. Proof — case studies, testimonials, data
7. Pricing — anchored, with risk reversal
8. FAQ — objection-first answers
9. Final CTA — repeat headline promise + direct CTA
10. Footer proof — logo bar, compliance badges, contact
```
